# 领域语言

本文档定义“本地化智能材料写作平台”前端、Java 后端和 Python AI 服务共同遵守的领域语言。后续接口、菜单、表格、表单、DTO、VO、Service、数据库字段和文档应优先使用本文术语。

## 术语总览

| 术语 | 推荐英文命名 | 含义 | 避免 |
|---|---|---|---|
| 材料 | `material` | 宣传文档的原始素材，由用户上传，由 Java 后端管理存储和元数据 | 资料、文档、document、resource |
| 片段 | `segment` | 材料经切分后生成的文本块，带有元数据，用于向量检索 | 分片、chunk |
| 来源片段 | `sourceSegment` | 一次检索命中并返回给写作任务或问答结果用于溯源展示的片段 | source、hit、检索结果 |
| 写作任务 | `writingTask` | 用户发起的一次内容生成请求，包含写作主题和期望输出类型 | 生成任务、请求 |
| 问答 | `qa`、`questionAnswer` | 用户基于材料提出问题并请求大模型回答的一次交互 | ask、QA、提问接口 |
| Prompt 模板 | `promptTemplate` | 用 Jinja2 编写的模板，根据写作任务和检索到的片段组装最终发送给大模型的提示词 | 提示词模板、prompt |
| 材料向量维护 | `materialVector` | Java 后端为保持材料与片段一致而调用 Python 服务写入、替换或删除向量的对接能力 | 知识库 CRUD、材料 CRUD |

## 材料

材料是宣传文档的原始素材，由用户上传，由 Java 后端管理存储和元数据。

避免使用：

- 资料
- 文档
- `document`
- `resource`

说明：

- Java 后端负责材料主数据、原始文件记录、权限、状态和生命周期。
- 文件上传只是材料进入系统的一种方式，不等于业务概念本身。
- 代码内部如必须使用英文，优先使用 `material`。

## 片段

片段是材料经切分后生成的文本块，带有元数据，用于向量检索。

避免使用：

- 分片
- `chunk`

说明：

- 片段由 Python AI 服务解析和切分产生。
- Java 后端可以保存片段元数据、处理状态或展示所需信息，但不应承担核心切分算法。
- 代码内部如必须使用英文，优先使用 `segment`。

## 来源片段

来源片段是一次检索命中并返回给写作任务或问答结果用于溯源展示的片段，包含正文、材料 ID、片段序号和相关度。

避免使用：

- `source`
- `hit`
- 检索结果

说明：

- 来源片段用于前端展示参考来源和溯源。
- 相关度可以用于排序、过滤、调试或低置信度提示。
- 相关度不一定直接展示给最终用户。
- 缺失材料 ID 或相关度时属于溯源降级，不应导致整个写作任务或问答失败。
- 代码内部如必须使用英文，优先使用 `sourceSegment`。

## 写作任务

写作任务是用户发起的一次内容生成请求，包含写作主题和期望输出类型。

输出类型包括但不限于：

- 提纲
- 初稿
- 润色稿
- 标题

避免使用：

- 生成任务
- 请求

说明：

- Java 后端负责写作任务的创建、状态、参数、结果保存和审计。
- Python AI 服务只负责根据 Java 传入的上下文生成内容。
- 代码内部如必须使用英文，优先使用 `writingTask`。

## 问答

问答是用户基于材料提出问题并请求大模型回答的一次交互。

避免使用：

- `ask`
- `QA`
- 提问接口

说明：

- Java 后端负责问答记录、用户权限、材料范围和结果保存。
- Python AI 服务负责检索和回答生成。
- 代码内部如必须使用英文，可使用 `qa` 或 `questionAnswer`，同一模块内保持一致。

## Prompt 模板

Prompt 模板是用 Jinja2 编写的模板，根据写作任务和检索到的片段组装最终发送给大模型的提示词。

避免使用：

- 提示词模板
- `prompt`

说明：

- Prompt 模板当前主要由 Python AI 服务使用。
- Java 后端可以管理模板配置或模板选择参数，但不要把模板拼接逻辑分散在多个层级。
- 代码内部如必须使用英文，优先使用 `promptTemplate`。

## 材料向量维护

材料向量维护是 Java 后端为保持材料与片段一致而调用 Python 服务写入、替换或删除向量的对接能力。

避免使用：

- 知识库 CRUD
- 材料 CRUD

说明：

- 材料主数据 CRUD 属于 Java 后端。
- 向量写入、替换、删除属于 Python AI 服务能力。
- Java 后端通过 `user_id + material_id` 控制向量维护范围。
- `material_id` 不承诺全局唯一，应结合 `user_id` 使用。
- 代码内部如必须使用英文，优先使用 `materialVector`。

## 命名约束

- 对外接口、菜单、表格、表单、DTO、VO、Service 命名，应尽量遵守本文术语。
- 数据库命名应避免把材料业务概念塞进 `sys_` 系统表前缀。
- 如果历史代码或第三方协议存在不一致命名，应在边界层做转换，并在文档中说明原因。
