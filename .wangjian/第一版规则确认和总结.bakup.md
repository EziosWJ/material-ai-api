# 后端系统业务模块设计总结

## 1. 项目基础定位

本项目后端定位为一个前后端分离的通用后台管理系统模板。

第一版目标：

- 服务现有 React 管理后台
- 采用 Java 21 + Spring Boot 3.x
- 采用 MySQL 8.0
- 采用 MyBatis-Plus
- 采用 Sa-Token 作为认证权限框架
- 采用模块化单体架构
- 单 Spring Boot 应用
- 单 Maven module
- 不使用微服务
- 不使用 Maven 多 Module
- 不使用 Flyway / Liquibase
- SQL 脚本手动维护

---

## 3. 权限模型

第一版权限模型采用：

```text
用户 -> 角色 -> 菜单
```

暂不实现：

- 按钮权限
- 接口权限
- 数据权限

菜单权限由后端根据当前登录用户角色过滤后返回。

前端只渲染后端返回的可见菜单。

## 4. 用户模块设计

### 4.1 用户基础能力

- 批量删除
- 启用 / 禁用
- 分配角色
- 所属部门
- 头像

### 4.2 用户字段设计

用户表建议包含：

```text
id
username
nickname
password
phone
email
avatar
gender
status
dept_id
last_login_time
last_login_ip
remark
create_time
update_time
create_by
update_by
deleted
```

### 4.3 已确认规则

- 用户名作为登录账号
- 用户名唯一
- 支持手机号
- 支持邮箱
- 手机号不要求唯一
- 邮箱不要求唯一
- 使用昵称字段，不单独设计真实姓名
- 需要头像
- 用户状态为启用 / 禁用
- 需要记录最后登录 IP
- 需要记录最后登录时间
- 密码必须加密存储

### 4.4 密码加密方式

密码加密建议使用 BCrypt。

规则：

- 数据库不保存明文密码
- 新增用户时加密密码
- 修改密码时重新加密
- 管理员重置密码时重新加密
- 登录时使用 BCrypt 校验明文密码与数据库密文是否匹配

不建议使用：

```text
MD5
SHA1
SHA256(password)
MD5(password + salt)
明文保存
可逆加密
```

推荐：

```text
BCrypt
默认强度 10 或 12
```

### 6.2 菜单类型

菜单支持：

```text

```

说明：

- 目录：用于侧边栏分组，例如“系统管理”
- 菜单：对应具体页面，例如“用户管理”
- 外链：跳转外部地址


### 6.4 已确认规则

- 菜单支持目录
- 菜单支持具体页面
- 菜单支持外链
- 菜单支持图标
- 菜单支持排序
- 菜单支持隐藏状态
- 菜单支持启用 / 禁用
- 菜单需要 component 字段
- 路由 path 由后端维护并返回给前端
- 内置菜单不允许删除
- 菜单需要权限标识字段 `permission_code`
- `permission_code` 唯一

### 6.5 React component 字段说明

React 不能直接根据后端字符串任意动态加载本地文件。

建议前端维护组件映射表，例如：

```ts
const componentMap = {
  "system/user/index": UserPage,
  "system/role/index": RolePage,
  "system/menu/index": MenuPage
}
```

后端返回：

```json
{
  "path": "/system/users",
  "component": "system/user/index"
}
```

前端根据 `component` 字段匹配本地组件。

## 14. 状态字段规范

所有启用 / 禁用状态统一使用：

```text
status tinyint
1 = 启用
0 = 禁用
```

不采用：

```text
status tinyint
Y / N
```

原因：

- `tinyint` 应使用数字值
- `Y / N` 更适合 `char(1)`
- 数字状态更适合筛选、排序、统计和多数据库适配
- 后续可以扩展更多状态

例如：

```text
0 = 禁用
1 = 启用
2 = 锁定
3 = 待审核
```
